# 二、题目解答

```c++
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
        // 我要从后向前插入不然nums1会经历向后的移位

        // 三个指针分别表明数组一二的正常有效数和新的最后一位有效数
        int i = m-1;
        int j = n-1;
        int k = m+n-1;
        
        while(i>=0 && j>=0){
            if(nums1[i]>nums2[j])
            {
                nums1[k] = nums1[i];
                i--;
            }
            else
            {
                nums1[k] = nums2[j];
                j--;
            }
            k--;
        }
        while(j>=0){
            nums1[k] = nums2[j];
            k--;
            j--;
        }
    }
};

```


# 三、面试中可能会遇到的问题


## 1. 为什么从后往前合并？

因为 `nums1` 后面有空位（`n` 个 `0` 占位符），从前往后合并会覆盖 `nums1` 中还未处理的有效元素。从后往前合并可以：利用 `nums1` 尾部的空闲空间、避免元素被提前覆盖、原地完成合并而不需要额外数组。

## 2. 为什么只处理 `while(j >= 0)`？

在代码中通常有两个循环：主循环处理两数组的比较，第二个循环单独处理 `nums2` 剩余元素。如果 `nums2` 已经处理完（`j < 0`），`nums1` 剩余的元素本来就在正确位置上，不需要移动；只有当 `nums2` 还有剩余元素时（`j >= 0`），才需要继续复制到 `nums1` 前面。

## 3. 时间复杂度是多少？

**O(m + n)**。每个元素（`nums1` 的 `m` 个有效元素 + `nums2` 的 `n` 个元素）最多只会被访问一次，两个指针从后向前移动，总共移动 `m + n` 步。

## 4. 空间复杂度是多少？

**O(1)**。只使用了常数个额外变量（三个指针），直接在 `nums1` 数组上进行原地修改，没有使用额外的数组空间。

## 5. 如果 `m = 0` 会怎么样？

此时指针 `i = -1`，主循环 `while i >= 0 and j >= 0` 不会执行，直接进入 `while j >= 0` 循环，将 `nums2` 全部复制到 `nums1` 中。例如：`nums1 = [0]`, `m = 0`, `nums2 = [1]`, `n = 1`，输出 `[1]`。

## 6. 如果 `n = 0` 会怎么样？

此时指针 `j = -1`，两个循环都不会执行，`nums1` 保持原样，无需任何操作。例如：`nums1 = [1,2,3,0,0,0]`, `m = 3`, `nums2 = []`, `n = 0`，输出 `[1,2,3,0,0,0]`，有效部分为 `[1,2,3]`。
