搭建个人博客是每个程序员都值得尝试的事情。这篇文章分享我的经验和思考。

## 技术选型对比

| 方案 | 优点 | 缺点 | 适合人群 |
|------|------|------|----------|
| 静态生成（Hexo/Hugo） | 速度快、免费托管 | 灵活性低 | 只写文章的人 |
| 动态博客（Express/Django） | 完全可控、可扩展 | 需要维护服务器 | 想练全栈的人 |
| Notion/语雀 | 零成本、开箱即用 | 不专业、SEO差 | 随便记点东西 |
| Ghost/WordPress | 功能完善 | 笨重、定制难 | 内容创作者 |

## 核心页面

```javascript
// 一个最小博客只需要三个路由
app.get('/', (req, res) => {
  res.render('index', { posts: getAllPosts() });
});

app.get('/post/:id', (req, res) => {
  const post = getPostById(req.params.id);
  post ? res.render('post', { post }) : res.status(404).send('404');
});

app.get('/about', (req, res) => {
  res.render('about');
});
```

## 必备功能清单

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 文章列表 + 详情 | 必须 | 博客的核心 |
| Markdown 渲染 | 必须 | 程序员最爱的写作格式 |
| 响应式设计 | 必须 | 手机端占比越来越高 |
| 搜索 | 选做 | 文章多了才需要 |
| 评论系统 | 选做 | 可以用 Giscus 等方案 |
| RSS 订阅 | 选做 | 技术博客的标配 |

> 博客的内容比博客本身更重要。用最简单的 HTML 写出好文章，也比用最酷炫的技术堆出空荡荡的页面有价值一百倍。

先写起来，再慢慢完善。你的第一篇博客不需要完美，它只需要存在。
